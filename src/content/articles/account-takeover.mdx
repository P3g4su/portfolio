## Introdução

Account Takeover (ATO) é uma das vulnerabilidades com maior impacto real em aplicações web. Ao contrário de exploits exóticos, a maioria dos ATOs acontece por falhas banais de validação.

Neste artigo vou mostrar como um único caractere — um `%00` — pode ser suficiente para comprometer toda uma conta.

## O cenário

Considere um sistema de recuperação de senha típico...

```python

def validate_token(token):
    return token in valid_tokens  # case-sensitive, mas e null bytes?
```
